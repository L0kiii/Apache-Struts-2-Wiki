Adding New FAQs

Each FAQ should be cross-referenced from a relevant page in one of the guides. (If a relevant page in the guide is missing, then we probably need to create one!) Each question should be a new page. Answers should be concise and focused. If an answer seems long, or seems like it could relate to more than one section, then the question might be addressing more than one concern.

FAQs can also be "mini-HOWTOs". As long as the question and answer are focused on a single concern, length is not an issue.

    Setting the constant "struts.action.extension" to an empty string
    makes all content be served by actions. This also applies to static
    content in the struts2 package like the JavaScript for the
    AJAX theme. To workaround the problem one has to override
    the DefaultActionMapper so that getMapping returns null for the static content.
    To do so you must set the constant "struts.mapper.class" to the classname that overrides DefaultActionMapper.

    See also http://www.mail-archive.com/user@struts.apache.org/msg64177.html
